### ¿Que es?

Es aquello que nos da permiso a los siguientes casos 

- Permiten acceder a servicios de Microsoft
- Definen que productos y servicios puede usar cada usuario
- Son obligatorias para cumplir con los contratos de uso y evitar sanciones

## Tipos

### Gratis

Esta licencia nos permite una administración básica de grupos y usuarios, sincronización de directorios locales, informes básicos, cambio de contraseñas etc

### P1
**Incluye todo lo de la anterior**

Acceso híbrido a recursos locales y en la nube, administración avanzada y grupos de pertenencia dinámicos, grupos de autoservicio y microsoft identity manager

### P2
**Incluye todo lo de las anteriores**

Microsoft Entra ID Protection (acceso condicional según riesgos), Privileged Identity Management (detectar, restringir y supervisar casos), Acceso just-in-time para administradores

### Governance

Gestión avanzada de identidades (para clientes P1 y P2), licencias de pago de uso

-------
### Comparativa entre planes P1 y P2

##### P1

Autoservicio de grupos -> Los usuarios crean, administran y solicitan unirse a grupos 

Informes y alertas avanzadas -> Detección de anomalías y patrones de acceso incoherentes

Autenticación multifactor -> Compatible con VPN, Azure, Microsoft 365, Dynamic 365

Microsoft Identity Manager -> Nos permite enlazar microsoft entra id con servicios locales (AD DS, LDAP, Oracle)

SLA 99,9%

Cloud App Discovery -> Detecta aplicaciones en la nube no autorizadas

Acceso condicional por dispositivo, grupo o ubicación

Entra Connect Health -> Monitoriza y entrega alertas sobre Entra ID

##### P2

Protección de Entra ID -> Podemos supervisar los inicios de sesión, directivas de riesgo de inicio de sesión y comportamiento de usuarios

Privileged Identity Management (PIM) -> Esto nos permite definir usuarios privilegiados, permite definir administradores temporales o permanentes y definir reglas y flujos de aprobación para elevar privilegios
